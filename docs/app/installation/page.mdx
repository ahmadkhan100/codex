# Installation

Get started with Codex CLI in just a few minutes. Follow our step-by-step installation guide for your operating system.

## Prerequisites

Before installing Codex CLI, ensure you have:

- Node.js 16.0 or higher
- npm or yarn package manager
- A valid OpenAI API key

## Quick Install

The fastest way to install Codex CLI is through npm:

```bash
npm install -g @openai/codex-cli
```

Or using yarn:

```bash
yarn global add @openai/codex-cli
```

## Platform-Specific Instructions

### macOS

```bash
# Using Homebrew
brew tap openai/tap
brew install codex-cli

# Or using npm
npm install -g @openai/codex-cli
```

### Windows

```powershell
# Using npm
npm install -g @openai/codex-cli

# Or using Chocolatey
choco install codex-cli
```

### Linux

```bash
# Using npm
npm install -g @openai/codex-cli

# Or using snap
snap install codex-cli
```

## Verify Installation

After installation, verify that Codex CLI is properly installed:

```bash
codex --version
```

You should see output similar to:

```
Codex CLI v1.0.0
```

## Configuration

### Set up your API key

1. Get your API key from [OpenAI Platform](https://platform.openai.com/api-keys)
2. Configure Codex CLI with your key:

```bash
codex auth login
```

Or set it as an environment variable:

```bash
export CODEX_API_KEY="your-api-key-here"
```

### Configure defaults

Set your preferred defaults:

```bash
# Set default model
codex config default_model gpt-4

# Set default output format
codex config output_format json

# Set request timeout
codex config timeout 60
```

## First Steps

Once installed and configured, try generating your first code:

```bash
codex generate "Create a function to reverse a string"
```

## Troubleshooting

### Common Issues

**Permission denied error**
```bash
# Fix with sudo (not recommended)
sudo npm install -g @openai/codex-cli

# Better: Fix npm permissions
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH
```

**API key not found**
```bash
# Check if key is set
codex auth status

# Re-authenticate
codex auth login
```

**Network timeout errors**
```bash
# Increase timeout
codex config timeout 120

# Check proxy settings
export HTTP_PROXY=http://proxy.example.com:8080
```
